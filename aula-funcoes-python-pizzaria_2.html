Funções em Python

Aprenda a estruturar, reutilizar e organizar código com funções, usando um sistema de pizzaria como fio condutor dos exemplos.

6 tópicos Python 3.x Graduação ADS def · return · escopo
01

Por que usar funções?

Imagine que você está desenvolvendo o sistema de pedidos da Pizzaria do Código. Toda vez que um cliente faz um pedido, você precisa calcular o valor total, aplicar desconto e gerar o recibo. Sem funções, você repetiria todo esse código a cada pedido.

♻️

Reutilização

Escreva uma vez, use para qualquer pedido. A lógica de desconto não precisa ser copiada 50 vezes.

🧩

Modularização

Separa calcular preço, aplicar desconto e gerar recibo em blocos independentes.

🐛

Facilidade de testes

Você pode testar o cálculo de desconto de forma isolada, sem precisar simular um pedido inteiro.

📖

Legibilidade

calcular_total_pedido() comunica a intenção muito melhor do que 10 linhas soltas.

❌ Sem funções
python 
# Pedido 1
preco1 = 45.90
desconto1 = preco1 * 0.10
total1 = preco1 - desconto1
print(f"Total: R$ {total1:.2f}")

# Pedido 2 — código repetido!
preco2 = 38.50
desconto2 = preco2 * 0.10
total2 = preco2 - desconto2
print(f"Total: R$ {total2:.2f}")
✅ Com funções
python 
def calcular_total(preco, desconto=0.10):
    return preco * (1 - desconto)

print(calcular_total(45.90))
print(calcular_total(38.50))
💡
Princípio DRY Don't Repeat Yourself — não se repita. Se você copia e cola código, provavelmente deveria criar uma função.
02

Estrutura de uma função em Python

Toda função em Python segue a mesma anatomia. Vamos dissecar uma função real do sistema da pizzaria:

ANATOMIA DA FUNÇÃO
def calcular_preco_pizza(sabor, tamanho, borda_recheada=False):
    "Calcula o preço final de uma pizza com opções."
    tabela = {"P": 25.0, "M": 35.0, "G": 45.0}
    preco = tabela[tamanho]
    if borda_recheada:
        preco += 8.0
    return preco

03

Parâmetros e retorno de funções

Parâmetros posicionais

A ordem dos argumentos importa. Trocar a posição muda o resultado:

Posicionais — a ordem importa python 
def montar_pizza(sabor, tamanho):
    print(f"Pizza de {sabor} tamanho {tamanho}")

montar_pizza("Calabresa", "G")   # Pizza de Calabresa tamanho G ✓
montar_pizza("G", "Calabresa")   # Pizza de G tamanho Calabresa ✗

Parâmetros com valor padrão (default)

Valores padrão são usados quando o argumento não é informado na chamada:

Valor padrão python 
def fazer_pedido(sabor, tamanho="M", borda_recheada=False):
    borda = "com borda recheada" if borda_recheada else "sem borda"
    print(f"Pedido: {sabor} | {tamanho} | {borda}")

fazer_pedido("Margherita")                     # M, sem borda
fazer_pedido("Frango", "G")                   # G, sem borda
fazer_pedido("Calabresa", "P", True)         # P, com borda recheada

Parâmetros nomeados (keyword arguments)

Ao nomear os argumentos, a ordem não importa mais — e o código fica mais legível:

Argumentos nomeados python 
def registrar_cliente(nome, telefone, endereco):
    print(f"Cliente: {nome} | Tel: {telefone} | End: {endereco}")

# Argumentos nomeados — ordem não importa
registrar_cliente(
    telefone="(11) 99999-0001",
    nome="Ana Lima",
    endereco="Rua das Pizzas, 42"
)

Retorno de valores

Retorno múltiplo — Python retorna uma tupla python 
def resumo_pedido(itens, desconto=0):
    subtotal = sum(itens)
    valor_desconto = subtotal * (desconto / 100)
    total = subtotal - valor_desconto
    return subtotal, valor_desconto, total  # retorna 3 valores

sub, desc, tot = resumo_pedido([35.0, 12.0, 8.5], desconto=10)
print(f"Subtotal:  R$ {sub:.2f}")   # 55.50
print(f"Desconto:  R$ {desc:.2f}")  # 5.55
print(f"Total:     R$ {tot:.2f}")   # 49.95
ℹ️
Sobre o return Uma função sem return retorna None implicitamente. Isso é esperado para funções que apenas realizam ações (como imprimir o cardápio), em vez de produzir valores.

04

Reutilização de código

Funções bem escritas são como as peças do sistema de pedidos da pizzaria: cada uma faz uma coisa, mas juntas formam um sistema completo. Veja como compor funções pequenas para resolver um problema maior:

Sistema de pedidos — composição de funções python 
# ── Funções pequenas e focadas ──────────────────────────

def calcular_preco_pizza(tamanho):
    tabela = {"P": 25.0, "M": 35.0, "G": 45.0}
    return tabela.get(tamanho, 0)

def calcular_taxa_entrega(distancia_km):
    if distancia_km <= 3:
        return 5.0
    elif distancia_km <= 7:
        return 10.0
    else:
        return 15.0

def aplicar_desconto_fidelidade(subtotal, eh_cliente_fiel):
    if eh_cliente_fiel:
        return subtotal * 0.90  # 10% de desconto
    return subtotal

def fechar_pedido(tamanho, distancia_km, eh_cliente_fiel=False):
    # Esta função REUTILIZA todas as anteriores!
    preco_pizza = calcular_preco_pizza(tamanho)
    taxa        = calcular_taxa_entrega(distancia_km)
    subtotal    = preco_pizza + taxa
    total       = aplicar_desconto_fidelidade(subtotal, eh_cliente_fiel)

    print(f"Pizza ({tamanho}):      R$ {preco_pizza:.2f}")
    print(f"Taxa entrega:     R$ {taxa:.2f}")
    print(f"Total:            R$ {total:.2f}")
    print("-" * 30)

# Reutilizando para diferentes pedidos
fechar_pedido("G", distancia_km=5, eh_cliente_fiel=True)
fechar_pedido("M", distancia_km=2)
fechar_pedido("P", distancia_km=10)
💡
Composição de funções fechar_pedido() não recalcula nada do zero — ela chama funções já existentes. Se a taxa de entrega mudar, basta alterar calcular_taxa_entrega() e todo o sistema é atualizado automaticamente.
Exercício — relatório de vendas diárias python 
def calcular_total_vendas(pedidos):
    return sum(pedidos)

def calcular_ticket_medio(pedidos):
    if not pedidos:
        return 0
    return calcular_total_vendas(pedidos) / len(pedidos)

def relatorio_diario(pedidos):
    total  = calcular_total_vendas(pedidos)
    ticket = calcular_ticket_medio(pedidos)
    print(f"Pedidos hoje:  {len(pedidos)}")
    print(f"Total vendido: R$ {total:.2f}")
    print(f"Ticket médio:  R$ {ticket:.2f}")

vendas_sexta = [45.0, 38.5, 62.0, 29.9, 55.0]
relatorio_diario(vendas_sexta)

Escopo de variáveis (local e global)

O escopo define onde uma variável existe e pode ser acessada. Em Python há dois escopos principais:

Escopo Global
taxa_servico = 0.10  # visível em todo o programa
Escopo Local — calcular_conta()
imposto = 0.05  # só existe aqui dentro
✓ pode ler taxa_servico (global)
✓ pode ler e escrever imposto (local)
✗ não pode acessar imposto aqui fora — ela é local da função
Escopo local vs global python 
nome_pizzaria = "Pizzaria do Código"  # variável GLOBAL

def exibir_recibo(total):
    mensagem_local = "Obrigado pela preferência!"  # LOCAL
    print(f"== {nome_pizzaria} ==")   # ✓ acessa global
    print(f"Total: R$ {total:.2f}")
    print(mensagem_local)

exibir_recibo(49.90)
# print(mensagem_local)  → NameError! não existe aqui fora
Usando a palavra-chave global (use com cuidado!) python 
total_vendas_dia = 0.0

def registrar_venda(valor):
    global total_vendas_dia  # "vou modificar a variável global"
    total_vendas_dia += valor

registrar_venda(45.90)
registrar_venda(38.50)
print(f"Total do dia: R$ {total_vendas_dia:.2f}")  # 84.40
🚫
Evite variáveis globais O uso de global torna o código difícil de rastrear e testar — qualquer função pode modificar o estado global a qualquer momento. Prefira sempre passar valores como parâmetros e receber resultados via return.
❌ Com global
python 
desconto_ativo = 10

def preco_com_desconto(preco):
    global desconto_ativo
    return preco * (1 - desconto_ativo / 100)
✅ Via parâmetro
python 
def preco_com_desconto(preco, desconto):
    return preco * (1 - desconto / 100)

# Chamada clara e testável
preco_com_desconto(45.0, 10)
06

Boas práticas em funções

01

Uma função, uma responsabilidade

Cada função deve fazer uma coisa e fazê-la bem. Se o nome tem "e" (calcular_e_imprimir_preco), divida em duas funções.

02

Nomes descritivos em snake_case

Use verbos que descrevem a ação: calcular_total(), verificar_estoque(). Evite f1(), func() ou fazCoisa().

03

Docstrings em funções públicas

Documente o que a função faz, seus parâmetros e o que retorna. Use """texto""" logo após o def.

04

Limite o número de parâmetros

Mais de 4–5 parâmetros é sinal de que a função está fazendo coisas demais. Considere agrupar dados em dicionários.

05

Evite efeitos colaterais desnecessários

Prefira funções que recebem dados, processam e retornam — sem modificar variáveis externas ou estados globais.

06

Funções curtas e focadas

Uma boa função cabe em uma tela (~20 linhas). Se precisar de scroll para lê-la, é hora de refatorar.

Exemplo de função bem escrita — com type hints e docstring python 
def calcular_troco(valor_pago: float, total_pedido: float) -> float:
    """
    Calcula o troco a ser devolvido ao cliente.

    Args:
        valor_pago:    Valor em reais entregue pelo cliente.
        total_pedido:  Valor total do pedido a ser pago.

    Returns:
        float: Valor do troco. Retorna 0.0 se o valor pago
               for menor que o total.

    Raises:
        ValueError: Se os valores informados forem negativos.
    """
    if valor_pago < 0 or total_pedido < 0:
        raise ValueError("Valores não podem ser negativos")
    if valor_pago < total_pedido:
        return 0.0
    return round(valor_pago - total_pedido, 2)

# Uso claro e legível
troco = calcular_troco(valor_pago=100.0, total_pedido=67.50)
print(f"Troco: R$ {troco:.2f}")  # Troco: R$ 32.50
ℹ️
Type Hints (Python 3.5+) As anotações : float e -> float não são obrigatórias, mas tornam o código muito mais legível e ajudam IDEs como VS Code e PyCharm a detectar erros antes da execução.

Resumo final

def

Declaração

Use def nome(params): para criar uma função.

parâmetros

Entrada

Posicionais, com default ou nomeados. Passam dados para a função.

return

Saída

Devolve um valor ao chamador. Pode retornar múltiplos valores.

escopo

Visibilidade

Local = dentro da função. Global = todo o programa. Prefira local.

DRY

Reutilização

Escreva uma vez, use em qualquer parte do sistema.

boas práticas

Qualidade

Uma responsabilidade, nome descritivo, docstring, sem global.
