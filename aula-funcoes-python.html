01

Por que usar funções?

Imagine ter que escrever o mesmo trecho de código toda vez que precisar calcular o IMC de um paciente em um sistema hospitalar. Funções resolvem exatamente esse problema.

♻️

Reutilização

Escreva uma vez, use em qualquer lugar. Evita duplicação e facilita manutenção.

🧩

Modularização

Divide o programa em partes menores e independentes, mais fáceis de entender.

🐛

Facilidade de testes

Você pode testar cada função isoladamente, encontrando bugs com mais facilidade.

📖

Legibilidade

Um nome de função bem escolhido comunica a intenção do código imediatamente.

exemplo motivador

❌ Sem funções

Python 
# Calcular área para 3 retângulos
base1, alt1 = 5, 3
area1 = base1 * alt1
print(area1)

base2, alt2 = 8, 4
area2 = base2 * alt2
print(area2)

base3, alt3 = 6, 7
area3 = base3 * alt3
print(area3)

✅ Com funções

Python 
def area_retangulo(base, altura):
    return base * altura

print(area_retangulo(5, 3))
print(area_retangulo(8, 4))
print(area_retangulo(6, 7))
💡

Princípio DRYDon't Repeat Yourself — não se repita. Funções são a principal ferramenta para seguir este princípio fundamental da programação.

03

Parâmetros e Retorno

Tipos de Parâmetros

Python parâmetros posicionais 
# Parâmetros posicionais — ordem importa!
def subtrair(a, b):
    return a - b

print(subtrair(10, 3))   # 7
print(subtrair(3, 10))   # -7  ← a ordem mudou o resultado!
Python parâmetros com valor padrão (default) 
# Valor padrão: usado quando o argumento não é informado
def saudar(nome, saudacao="Olá"):
    print(f"{saudacao}, {nome}!")

saudar("Ana")                    # Olá, Ana!
saudar("Carlos", "Boa tarde")  # Boa tarde, Carlos!
Python parâmetros nomeados (keyword arguments) 
# Passando argumentos pelo nome — ordem não importa!
def criar_usuario(nome, email, idade):
    print(f"Usuário: {nome}, {email}, {idade} anos")

criar_usuario(email="ana@ads.com", nome="Ana", idade=22)
o return

Retorno de Valores

Python retorno múltiplo 
# Python permite retornar múltiplos valores!
def dividir(dividendo, divisor):
    quociente  = dividendo // divisor
    resto      = dividendo  % divisor
    return quociente, resto  # retorna uma tupla

q, r = dividir(17, 5)
print(f"Quociente: {q}, Resto: {r}")  # Quociente: 3, Resto: 2
🔵

DicaUma função sem return retorna None implicitamente. Isso é diferente de um erro — é comportamento esperado para funções que só realizam ações (como imprimir).

resumo
Tipo de parâmetro Sintaxe Exemplo
Posicional def f(a, b) f(1, 2)
Valor padrão def f(a, b=10) f(1) ou f(1, 5)
Nomeado def f(a, b) f(b=2, a=1)
Arbitrário (*args) def f(*args) f(1, 2, 3, 4)
04

Reutilização de Código

Funções bem escritas são como peças de LEGO: encaixam em qualquer contexto e podem ser combinadas para construir sistemas complexos.

Python sistema de notas — composição de funções 
# Funções pequenas e reutilizáveis
def calcular_media(notas):
    return sum(notas) / len(notas)

def situacao(media):
    if media >= 7.0:
        return "Aprovado ✅"
    elif media >= 5.0:
        return "Recuperação ⚠️"
    else:
        return "Reprovado ❌"

def relatorio_aluno(nome, notas):
    # Esta função REUTILIZA as outras!
    media  = calcular_media(notas)
    status = situacao(media)
    print(f"Aluno: {nome} | Média: {media:.1f} | {status}")

# Reutilizando para vários alunos
relatorio_aluno("Ana",    [8.0, 7.5, 9.0])
relatorio_aluno("Bruno",  [5.0, 4.5, 6.0])
relatorio_aluno("Carla",  [3.0, 4.0, 2.5])
✅

Composição de funçõesFunções podem chamar outras funções. Isso permite construir sistemas complexos a partir de peças simples — cada uma faz apenas uma coisa, mas juntas resolvem problemas grandes.

Python exercício: calculadora simples 
def somar(a, b):       return a + b
def subtrair(a, b):   return a - b
def multiplicar(a, b): return a * b
def dividir(a, b):
    if b == 0:
        return "Erro: divisão por zero"
    return a / b

def calcular(op, a, b):
    operacoes = {
        '+': somar,
        '-': subtrair,
        '*': multiplicar,
        '/': dividir
    }
    func = operacoes.get(op)
    if func:
        return func(a, b)
    return "Operação inválida"

print(calcular('+', 10, 5))   # 15
print(calcular('/', 10, 0))   # Erro: divisão por zero

Escopo de Variáveis

O escopo define onde uma variável pode ser acessada. Em Python existem dois principais: global e local.

escopo global
x = 10  # visível em todo o programa
escopo local — função minha_funcao()
y = 20  # só existe dentro desta função
✓ pode ler x (global)
✓ pode ler e escrever y (local)
✗ não pode ler y aqui fora (local da função)
Python escopo local vs global 
contador = 0  # variável GLOBAL

def incrementar():
    local_var = 100  # variável LOCAL
    print(f"Dentro da função: {contador}")   # lê global ✓
    print(f"Var local: {local_var}")

incrementar()
print(f"Fora da função: {contador}")    # 0
# print(local_var)  ← NameError! local_var não existe aqui
Python usando a palavra-chave global 
total = 0

def adicionar(valor):
    global total  # "aviso": vou modificar a variável global
    total += valor

adicionar(5)
adicionar(3)
print(total)  # 8
🚫

Evite variáveis globaisO uso de global torna o código difícil de rastrear e testar. Prefira sempre passar valores como parâmetros e receber via return.

❌ Evitar

Python 
desconto = 0.1

def calcular_preco(preco):
    global desconto
    return preco * (1 - desconto)

✅ Preferir

Python 
def calcular_preco(preco, desconto):
    return preco * (1 - desconto)
06

Boas Práticas em Funções

01

Uma função, uma responsabilidade

Cada função deve fazer uma coisa bem feita. Se o nome tiver "e" (ex: "calcular_e_imprimir"), provavelmente deve ser dividida.

02

Nomes descritivos em snake_case

Use verbos que descrevem a ação: calcular_media(), verificar_login(). Evite f1() ou fazCoisa().

03

Docstrings em funções públicas

Documente o que a função faz, seus parâmetros e o que retorna. Use """texto""" logo após o def.

04

Limite o número de parâmetros

Mais de 4-5 parâmetros é sinal de que a função está fazendo coisas demais. Considere agrupar dados em dicionários ou criar uma classe.

05

Evite efeitos colaterais desnecessários

Prefira funções que recebem dados, processam e retornam um resultado — sem modificar variáveis externas ou estados globais.

06

Mantenha funções curtas e focadas

Uma boa função cabe em uma tela (~20 linhas). Se precisar de scroll para lê-la, considere refatorar.

Python exemplo de função bem escrita 
def calcular_imc(peso_kg: float, altura_m: float) -> float:
    """
    Calcula o Índice de Massa Corporal (IMC).

    Args:
        peso_kg:  Peso em quilogramas.
        altura_m: Altura em metros.

    Returns:
        float: O valor do IMC calculado.

    Raises:
        ValueError: Se altura ou peso forem negativos ou zero.
    """
    if peso_kg <= 0 or altura_m <= 0:
        raise ValueError("Peso e altura devem ser positivos")
    return peso_kg / (altura_m ** 2)

# Uso claro e legível
imc = calcular_imc(peso_kg=70, altura_m=1.75)
print(f"IMC: {imc:.1f}")  # IMC: 22.9
🔵

Type Hints (Python 3.5+)As anotações de tipo : float e -> float não são obrigatórias, mas tornam o código muito mais legível e ajudam ferramentas como IDEs e linters a detectar erros.

Resumo Final

def

Declaração

Use def nome(params): para criar uma função.

parâmetros

Entrada

Posicionais, com default ou nomeados. Entrada de dados para a função.

return

Saída

Devolve um valor. Pode retornar múltiplos valores como tupla.

escopo

Visibilidade

Local = dentro da função. Global = todo o programa. Prefira local.
